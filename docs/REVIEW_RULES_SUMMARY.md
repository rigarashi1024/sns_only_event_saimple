# Review Rules Summary

このファイルは Gemini PR Review に渡すための短いルール要約です。

## 基本方針

- 重大な問題のみを指摘する
- 対象は `bug`, `security`, `logic` のみ
- スタイル、命名、フォーマット、好みの指摘はしない
- patch に含まれない情報を推測して断定しない
- 情報不足なら問題として報告しない

## このリポジトリで重視する観点

- イベント駆動の流れを壊していないか
- 冪等性を壊していないか
- 認証、トークン、暗号化まわりに危険がないか
- Firestore の document id 設計や重複防止前提を壊していないか
- OpenAPI と実装の責務分離を壊していないか

## 報告しないもの

- README やコメントだけの変更
- 任意のリファクタ提案
- 軽微な最適化
- 個人の好みによる設計論
